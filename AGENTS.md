# AGENTS

This repository is a Hugo website built with the `hugo-coder` theme.

## Workspace agent guidance

Use `.github/copilot-instructions.md` for the main workspace-level instructions.

## Primary focus areas

- `content/` — markdown content and blog posts.
- `config.toml` — site metadata, menus, theme settings, and global parameters.
- `layouts/` — Hugo templates, layouts, and partials.
- `assets/` — SCSS styling, JavaScript, and static build assets.
- `static/` — published static files such as images and site assets.

## Generated output

- `public/` is generated by `hugo` and should not be edited directly.
- `resources/` is Hugo build cache and should not be edited.

## Useful commands

- `git submodule update --init --recursive`
- `hugo server --buildDrafts --buildFuture`
- `hugo`

## Notes

- In this repo, the Hugo theme is stored as a submodule at `themes/hugo-coder/`.
- There is a `workflows/hugo.yaml` file in the repo, but no `.github/workflows` directory.
- `config.toml.bak.toml` is a backup configuration file.

## Example prompts

- "Update the blog navigation so the menu includes About, Blog, Projects, and Contact."
- "Add a new post under `content/posts/` with metadata for title, date, and tags."
- "Fix the broken image path in `static/` and update the corresponding post front matter."
- "Change the site color scheme to light mode in `config.toml`."
